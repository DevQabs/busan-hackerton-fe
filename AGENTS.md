# AGENTS.md — AI agent handoff guide

You are working on **어디든 두가자**, a DIVE 2026 hackathon dashboard (finals
2026-07-25/26, BEXCO): Busan 교통약자 trip demand (두리발) crossed with
barrier-free (무장애) infrastructure to find 접근성 사각지대 — high-demand /
low-infrastructure 행정동 — and rank improvement priorities.

Read this file first. Then: `lib/types.ts` (the data contract),
`pipeline/README.md` (data quirks + formulas), root `README.md` (run guide).

## Architecture — one-way data flow

```
raw CSVs/xlsx/SHP/zip  →  pipeline/build_all.py  →  public/data/*.json  →  Next.js UI
       (inputs)              (pure-stdlib py)       (CONTRACT, frozen)      (reads ONLY /data)
```

- **`lib/types.ts` is the frozen contract.** The pipeline writes artifacts that
  match it; the UI renders them. Change it only when you update BOTH sides in
  the same commit, and keep it backward-compatible during the finals.
- **Never hand-edit `public/data/*.json`** (except `model_results.json`, see
  below). They are regenerated by the pipeline; manual edits will be lost.
- The UI has no backend, no DB, no API routes. Keep it that way — the demo
  must run offline from static files.

## Methodology — what is real vs placeholder

Currently REAL (computed from May 2025 public 두리발 data, citywide):
- Trip categories (disjoint, sum = 37,512): completed 32,526 (`결과=하차`),
  unassigned 3,520 (`결과=미배차`), cancelled 1,466 (`상태` 취소-family).
- Wait = 접수→승차 minutes, completed trips only, 0–24h window
  (median 42.5 / p90 87.3). CAVEAT: 예약-type trips inflate this; consider
  recomputing on `예약타입=='즉시'` only (99.5% of trips anyway).
- Per-dong joins: trips by 행정동-name matcher (100% Busan-side; handles
  가야1동↔가야제1동 and 일광면→일광읍), shops by 행정동코드 8-digit == adm_cd2[:8],
  POIs by point-in-polygon.

PLACEHOLDER (to be replaced by the DS at the finals — see team stat guide):
- `gapScore = z(dropoffs) − infraZ + 0.5·z(unassigned)` where
  `infraZ = mean(z(chargers), z(hospitals+pharmacies), z(welfare), z(floor1-weighted shops))`.
  Target replacement: Negative Binomial regression
  `하차건수 ~ 인프라지수 + log(상가수) + log(장애인구)`, report IRR with 95% CI.
- `gapClass`: H = top tercile. HL (high demand, low infra) = priority = red.
- The "충전소 +1" simulation in the UI recomputes gapScore client-side with an
  embedded sigma — an explicitly labeled approximation.

## Finals data swap (the whole point of this design)

1. New inputs arrive (해운대구-only, 43,891 rows, **11 columns — destination
   coordinates only, no origin coords**). Edit paths at the top of
   `pipeline/build_all.py`.
2. The pipeline must degrade gracefully: OD arcs and pickup-side layers need
   origin coords — if absent, emit empty `od.json` / `unmet.json` (the UI
   already shows "데이터 준비 중" for empty/missing artifacts).
3. Re-run build + validate. UI code should need ZERO changes.
4. Privacy (organizer rules): finals data coordinates may only be shown
   aggregated (행정동 / 격자 / rounded cells). `unmet.json` already rounds to
   ~100m cells; keep any new layer aggregated the same way. No individual
   trip tracking, no re-identification.

## Data quirks that WILL bite you (full table: pipeline/README.md)

- 두리발 CSV: CP949; `X좌표` is LATITUDE, `Y좌표` is LONGITUDE (swapped —
  verify by range, lat≈35, lng≈129). HIRA xlsx is NOT swapped (좌표(X)=lng).
- Chargers CSV: filter Busan by `시도명=='부산광역시'`, never by address
  (부산면 in 전남 false-positives).
- 상가 zip: entry names are UTF-8(NFD) mangled through cp437 — decode
  `name.encode('cp437').decode('utf-8')` + NFC-normalize; stream, don't extract.
- Toilets/metro-stations CSVs have NO coordinates (gu-level only / name join).
- Dong GeoJSON: vuski/admdongkor ver20260701, 206 features, key = `adm_cd2`.

## Known issues (accepted, do not "fix" silently)

- Basemap (basemaps.cartocdn.com) is blocked on some corporate networks; the
  UI falls back to layers-on-dark-bg with an "오프라인" badge. Test once on
  normal internet before the demo.
- 4 welfare POIs sit outside every dong polygon (coastline) — kept in
  `infra_points.json` without `dong`, excluded from per-dong counts.
- 1 미배차 row lacks pickup coords → unmet cells sum to 3,519 (stats.json
  keeps the true 3,520). 5 toilet rows have unparseable 구.
- `useData` re-polls missing artifacts every 8s by design (lets the pipeline
  write files while the UI is running).
