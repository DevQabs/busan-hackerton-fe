# DIVE 2026 본선 전략 — 어디든 두가자 (아마란스H)

> 작성 배경: 5개 관점(방법론·최적화·도메인·데모·실행가능성)에서 30개 아이디어를 생성하고,
> 제약 조건(본선 데이터 11컬럼·출발지 좌표 없음 / 오프라인 데모 / 준비 기간 실질 12~14 person-day /
> 개인정보 규정 / 공공기관 심사위원)에 대해 적대적 검증을 거쳐 선별한 결과다.
> 점수는 "학생 팀 기준선 대비 기대 득점 × 실현 가능성" 추정치(0~100)다.

---

## 1. 전제 — 우리가 놓인 판

- **11개 팀이 같은 과제**를 받는다. 대부분 학생 팀. 예상 기준선: 수요 choropleth + 인프라 오버레이
  + 상관/단순회귀 + "여기에 지으세요" + (아마도) 챗봇. Streamlit/Kepler 계열 화면.
- **심사 배점**: 문제정의 20 / 창의성·차별성 20 / **데이터 기반 문제해결 30** / 구현 완성도 20(실제 활용
  가능성 기준) / 발표 10.
- **본선 데이터 제약**: 해운대구 한정, 43,891행 × 11컬럼. 타임스탬프 6개 + **목적지** 주소·행정동·좌표 +
  휠체어 여부. **출발지 좌표 없음** → OD 분석·출발지 기반 레이어는 본선 데이터로는 재현 불가.
  좌표는 행정동/격자 단위로만 집계 표시(개인 이동 추적 금지).
- **팀 구성**: 프론트엔드(React 전문) / 백엔드·인프라 / 데이터 사이언티스트 / 제너럴리스트.
- **이미 구축됨**: Next.js + deck.gl 대시보드(11개 씬), 순수 파이썬 파이프라인, 고정 JSON 계약
  (`lib/types.ts`). 5월 시 전역 공공 데이터(37,512건)로 리허설 완료.

## 2. 핵심 전략 — 세 개의 "높이 이동"

모든 팀이 같은 평면(공간 지도)에서 싸운다. 우리는 세 축으로 이동한다:

1. **시간축**: 타임스탬프 6개는 본선 데이터에서 유일하게 확실히 살아남는 자산이다.
   대기시간의 생존분석, 대기 구간 분해, 차량 가동률 — 지도가 아니라 시계열에서 이긴다.
2. **문 앞(door-level)**: 행정동 choropleth가 아니라 하차 지점 250m 격자 × 시설별 도달 거리.
   "이 격자에 내리는 휠체어 이용자가 한 달에 N명인데 300m 안에 경사로 있는 가게가 없다"는
   문장은 구청이 바로 쓸 수 있는 언어다.
3. **부재(不在)의 지도**: 관측된 수요가 아니라 **관측되지 않은 수요**. 등록 장애인은 많은데
   이용이 비정상적으로 적은 동 = 침묵 지역. 나머지 10개 팀의 내러티브를 뒤집는다.

## 3. MUST — 반드시 구현 (합계 ≈ 10 person-day)

### 3-1. 대기시간 포렌식 (82점) — 담당: DS, 2.5pd

**무엇**: ① Kaplan-Meier 생존곡선으로 대기시간 추정(미배차·취소를 관측중단(censored)으로 포함),
② 대기 구간 분해: 접수→배차 vs 배차→승차, 시간대별 p50/p90, ③ 배차→하차 구간 겹침으로
시간대별 차량 가동률 곡선.

**왜 차별화인가**: 공식 지표 "대기 중앙값 42분"은 **배차에 성공한 사람만의 통계**다.
미배차 9.4%는 분모에서 사라진다. 생존분석은 이들을 포함해 진짜 대기 부담을 추정한다 —
심사석에 앉은 공단 실무자의 KPI를 한 단계 업그레이드해 주는 것이고, 학생 팀은 여기까지
내려가지 않는다.

**전달 톤(중요)**: "귀 기관의 지표가 틀렸다"가 아니라 **"기존 지표를 보완하는 추정을 더했다"**.
심사위원이 곧 지표의 주인이다.

**리스크와 대비**: 본선 11컬럼에 상태 플래그가 없으면 미배차는 null 타임스탬프로 식별해야 한다.
그런 행이 아예 없으면 → 장기 대기(p90 초과)를 절단점으로 한 대안 분석으로 전환. Day 1 확인 항목.

### 3-2. 도착지 사각지대 + last-100m 커버리지 (76점) — 담당: DS+백엔드, 3pd

**무엇**: 하차 지점 250m 격자 스코어링. 격자별 하차 수 × 시설 부족 가중치(충전소/병의원/복지시설
도달 거리, 1층 상가 비율). 상위 격자 순위표 + 각 행에 조치 소관(공단/구청/윌체어) 배지.
greedy maximal-coverage로 "다음 K개 시설 후보 지점"과 한계 커버리지 곡선(다섯 번째 지점까지는
크게, 여섯 번째부터 +2%) 제시.

**왜 차별화인가**: 과제 문안("이동수요 × 무장애 인프라 융합")의 가장 문자 그대로의 답을
**문 앞 높이**에서 실행한 것. 본선에서 윌체어의 무장애가게 데이터(12개 Y/N 필드: 경사로·입구턱·
장애인화장실 등)가 오면 "휠체어 이용자가 자주 내리는데 입구턱 있는 가게만 있는 곳" 같은
가게 단위 결론으로 내려간다. greedy의 서브모듈러 보장(최적해의 63% 이상)은 무대에서 말할 수
있는 한 줄이다.

**리스크와 대비**: 리허설 단계에선 공공 데이터(충전기·병의원·복지시설·1층 상가 비율)로 구동,
본선에서 윌체어 레이어로 교체. 후보 지점은 공공 용도 건물(관공서/복지/의료)로 필터 —
사유 건물 주소를 지목하는 정치적 부담 회피.

### 3-3. 잠재수요(침묵 지역) 지수 (74점) — 담당: DS, 2pd

**무엇**: 음이항 회귀에 **노출(exposure) 오프셋**을 넣는다:
`하차건수 ~ 인프라 변수들, offset = log(장애인등록 인구)` (리허설에선 프록시로 log(상가수)).
편차 잔차(deviance residual)를 표준화 → 강한 음수 = "등록 인구 대비 이용이 비정상적으로 적은 동"
= 잠재수요 의심 지역. 지도에서 관측 수요 ↔ 잠재수요 토글.

**왜 차별화인가**: 관측 수요 기반 사각지대 논리에는 선택 편향이 있다 — 인프라가 아예 없는 곳은
갈 수 없어서 수요도 낮게 **관측**된다. 이 지수는 그 편향을 정면으로 다루며, "출발지 좌표 없음"
이라는 본선 데이터의 약점을 방법론적 강점으로 바꾼다(목적지 수요만으로 성립). 이미 계획된
NB 회귀를 재사용하므로 한계 비용이 낮다.

**리스크와 대비**: 중력모형 보정은 도착지 데이터만으로는 식별 불가 — 본 발표에서 제외하고
오프셋-잔차만 주장한다(방어 가능). 프록시 노출은 화면에 명시("본선에서 장애인등록 인구로 교체").

### 3-4. Before/After what-if — 모델 구동 시뮬레이션 (72점) — 담당: FE, 2.5pd

**무엇**: 기존의 단순 "+충전소 1개" 시뮬레이션을 NB 모델의 IRR로 교체. 동 선택 → 시설 추가 →
모델 예측 방문 변화 + **95% 신뢰구간을 화면에 그대로 표시** + 순위 변화.

**왜 차별화인가**: Streamlit 팀은 모델과 UI가 물리적으로 분리돼 있다. 우리는 고정 JSON 계약으로
모델 출력이 UI를 직접 구동한다. 신뢰구간을 숨기지 않는 것 자체가 심사위원(실무자) 앞에서
성숙함의 신호다.

**필수 멘트**: "연관성 기반 추정이며 인과가 아니다 — 그래서 구간을 함께 보여드린다."
IRR이 유의하지 않게 나오면 0을 포함한 구간을 정직하게 표시하고 무게를 3-2로 옮긴다.

## 5. 하지 않기로 한 것 (Q&A에서 잃는 게 더 많음)

- **차량 큐 DES 시뮬레이터**: 가정 공격 표면이 넓고, 가동률 곡선(3-1에 포함)이 같은 메시지를 더 싸게 전달.
- **Moran's I / LISA**: 행정동 n≈18(해운대)에서 통계적으로 무의미. 질문 방어용 한 줄로만 보유.
- **시민 앱 phone-frame**: 과제는 정책 대시보드다. 컨셉 화면 하나로 족함.
- **페르소나 리플레이("김OO님의 하루")**: 개인정보 규정과 정면 충돌 소지. 유령 씬이 같은 감정을 안전하게 전달.
- **LLM 실시간 생성(브리프/챗봇)**: 행사장 네트워크 리스크 대비 효과 미미. 사전 생성 텍스트로 대체.
