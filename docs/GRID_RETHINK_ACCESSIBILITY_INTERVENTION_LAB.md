# 도착 이후 400m — The Last 400m

**두리발은 사람을 데려다줍니다. 그러나 "내렸다"가 "들어갈 수 있다"는 아닙니다.**
우리는 하차 이후 실제로 들어갈 수 있는 곳이 몇 개인지, 어디서 막히는지, 무엇을
고쳐야 하는지를 보여줍니다. 슬로건: `도착은 접근이 아니다`.

## 문제

다른 팀은 「인프라가 부족한 곳」을 빨간 지도로 보여줄 것입니다. 하지만 빨간 점수는
조치가 아닙니다 — 충전기를 놓을지, 입구를 고칠지, 아무것도 안 할지 말해주지 않습니다.

## 우리가 하는 것

두리발 **하차 지점** × 윌체어 **무장애가게 12개 실사**(경사로·입구턱·엘리베이터·
장애인화장실 등)를 겹칩니다.

1. **enterable / usable** — 하차 근처에서 실제로 들어가 이용할 수 있는 가게 수.
2. **Barrier DNA** — 사슬 `진입 → 이용 → 편의` 중 어디서 끊기는지.
3. **다음 행동** — 무엇을(입구 무턱화 / 장애인화장실 설치), 누가(구청·건물주 / 업주),
   몇 건의 하차에 닿는지.

점수 하나가 아니라 **가장 먼저 끊기는 고리**로 판정합니다(weakest-link). 12개 중
11개가 좋아도 입구 문턱 하나면 접근은 0입니다.

## 화면

- **지도** — 하차(파란 점) 위에 무장애가게: 🔴 진입 불가 · 🟡 진입 가능·미완비 ·
  🟢 완비.
- **이동완성 깔때기** — 무장애가게 → 진입 가능 → 이용 가능 → 완비, 선택지가 줄어드는
  모습.
- **Barrier DNA** — 사슬이 끊기는 지점 분포.
- **다음 행동** — 무엇을·누가·몇 건, 행을 누르면 지도로 이동.
- **before / after** — 「입구를 고치면」 진입 가능 가게가 늘어나는 시뮬.

## 실제 결과 (송정동 표본)

- 두리발 하차 **1,129건**, 무장애가게 **21곳**.
- 진입 가능 **18**, 내부 이용 **17**, **완비 0**.
- **장애인화장실 보유 0곳**, 입구에서 막힘 3곳.

> 「가게가 없는 게 아니라, 문 앞·화장실에서 끊긴다.」

## 왜 이기나

- 타 팀은 「어디가 부족한가」. 우리는 **무엇을·누가·몇 건** — 검증 가능한 실행 목록.
- 윌체어 무장애가게 실사(좌표 + 12개 Y/N)를 이 방식으로 쓰는 건 우리뿐입니다.
- unknown은 빨간색이 아니라 **조사 과제**로 다룹니다.

## 90초 시연

1. 동 평균은 정상으로 보임 →
2. 하지만 이 미시존은 하차는 많은데 완비 0 →
3. Barrier DNA: 입구·화장실에서 끊김 →
4. `다음 3개 행동`을 누르면 접근 선택지 수가 바뀜 →
5. 격자 크기·기간을 바꿔도 우선순위 유지(확신) →
6. 구청이 고치고, 공단이 근거를 대고, 윌체어가 실사를 갱신.

## 데이터 계약

`public/data/access_actions.json` — 가게별 좌표·12개 항목·판정·인근 하차 + 요약.
UI는 이 파일만 읽습니다. 본선 전체 해운대 데이터로 재생성하면 씬은 그대로 확장됩니다.

## 주의

- 거리는 직선 proximity 기준 — 언덕·횡단보도 미반영. `wheelchair reachable`이라
  하지 않습니다.
- 현재는 송정동 표본(21곳). 이용자 수·이동 증가는 약속하지 않습니다.
